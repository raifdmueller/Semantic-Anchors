== Qualitätsanforderungen

=== Qualitätsbaum

```
Qualität
├── Usability (Prio 1)
│   ├── Findability: Anker in max. 3 Klicks finden
│   ├── Learnability: Neue User verstehen UI in < 2 Min
│   └── Efficiency: Filter/Search < 300ms Response
│
├── Performance (Prio 2)
│   ├── Load Time: < 2s on 3G
│   ├── Time to Interactive: < 3.5s
│   └── Card Grid Rendering: < 500ms
│
├── Maintainability (Prio 3)
│   ├── Modularity: Neue Anker hinzufügen in < 10 Min
│   ├── Testability: 80% Code Coverage
│   └── Analyzability: arc42 + ADRs aktuell
│
├── Accessibility (Prio 4)
│   ├── WCAG 2.1 AA Compliance
│   ├── Keyboard Navigation: Alle Funktionen erreichbar
│   └── Screen Reader: 100% Compatibility
│
└── Portability (Prio 5)
    ├── Browser Support: Chrome, Firefox, Safari, Edge (last 2 years)
    └── Deployment: < 30 Min auf neue Plattform migrierbar
```

=== Qualitätsszenarien

==== Usability

**QS-U1: Schnelles Finden eines Ankers**

[cols="1,3", options="header"]
|===
|Feld |Wert

|Szenario
|Software Developer sucht Anker "TDD, London School"

|Stimulus
|Öffnet Website zum ersten Mal

|Artefakt
|Semantic Anchors Website

|Umgebung
|Desktop Browser, normale Internetverbindung

|Antwort
|1. Nutzt Suchbox, tippt "TDD" +
2. Sieht Ergebnisse in < 300ms +
3. Klickt auf "TDD, London School" +
4. Sieht Details

|Metrik
|≤ 3 Klicks, ≤ 30 Sekunden Gesamtzeit
|===

==== Accessibility

**QS-A1: Keyboard-Navigation**

[cols="1,3", options="header"]

|===

=== Continuous Quality Monitoring

**GitHub Actions Checks:**

```yaml
- name: Lighthouse CI
  run: lhci autorun

- name: axe Accessibility Tests
  run: npm run test:a11y

- name: Unit Tests with Coverage
  run: npm run test:coverage

- name: E2E Tests
  run: npm run test:e2e
```

**Quality Gates:**

* Lighthouse Performance: > 90
* Lighthouse Accessibility: 100
* Unit Test Coverage: > 80%
* E2E Tests: All passing
* Bundle Size: < 500KB

**Automated Checks on PR:**

* Lighthouse CI läuft auf Preview Deploy
* axe-core scannt alle Seiten
* Performance Budget Check (Bundle Size)
